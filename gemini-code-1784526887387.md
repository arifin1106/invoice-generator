# Product Requirements Document (PRD)

**Nama Produk:** Web-Based Invoice Generator (School/Institution Edition)  
**Platform:** Web Application  
**Tech Stack:** React.js (Frontend) & Laravel (Backend)  
**Versi:** 1.0  

---

## 1. Ringkasan Eksekutif
Aplikasi ini adalah platform berbasis web yang memungkinkan staf administrasi atau keuangan untuk membuat, mengelola, dan mencetak *invoice* tagihan secara digital. Aplikasi ini dirancang khusus untuk memfasilitasi pembuatan *invoice* yang terstandarisasi, rapi, dan dapat diekspor ke dalam format PDF sesuai dengan format desain referensi.

## 2. Tujuan Produk
* Menyediakan antarmuka yang intuitif bagi pengguna untuk memasukkan data tagihan.
* Mengotomatisasi perhitungan total tagihan, pembayaran yang diterima, dan sisa tagihan.
* Menghasilkan *invoice* berformat PDF yang siap dikirimkan kepada klien/wali murid.
* Menyimpan riwayat *invoice* untuk keperluan pencatatan dan audit keuangan.

## 3. Spesifikasi Teknis (Tech Stack)
* **Frontend:** React.js (dengan Vite/Create React App), Tailwind CSS (untuk *styling*), Axios (untuk *HTTP requests*), dan pustaka manajemen *state* seperti Redux atau Zustand.
* **Backend:** Laravel (PHP), menyediakan RESTful API.
* **Database:** MySQL atau PostgreSQL.
* **PDF Generator:** *Library* pada backend seperti `barryvdh/laravel-dompdf` atau *library* frontend seperti `react-pdf` / `html2canvas` + `jspdf`.

## 4. Kebutuhan Fitur (Feature Requirements)

### 4.1. Manajemen Invoice (Frontend & Backend)
* **Form Pembuatan Invoice (Create Invoice):**
  * Input data instansi (bisa diatur *default* atau diubah).
  * Input data klien/siswa (Nama, Level/Kelas).
  * Input detail *invoice* (Nomor Invoice, Tanggal, Jatuh Tempo).
  * Tabel dinamis untuk *Item Tagihan* (pengguna bisa menambah/menghapus baris tagihan).
  * Otomatisasi kalkulasi: `Total`, `Bayaran Diterima`, dan `Sisa Tagihan`.
  * Input informasi tambahan: Pesan khusus, Detail Pembayaran (Bank), dan Penandatangan.
* **Daftar Invoice (Invoice List):** Menampilkan tabel riwayat *invoice* yang pernah dibuat, lengkap dengan status, tanggal, dan fitur pencarian/filter.
* **Aksi Invoice:** Preview (Pratinjau), Edit, Delete, dan Download PDF.

### 4.2. Kebutuhan Data & Form Field
Sistem harus menyediakan form dengan *field* berikut:

**A. Header & Informasi Instansi**
* Logo Instansi (Upload image atau gunakan *default* sistem).
* Nama Instansi (Contoh: Jakarta Cosmopolite Islamic School).
* Alamat Lengkap.
* Nomor Telepon & Email.

**B. Meta Invoice & Data Siswa/Klien**
* **Invoice Number:** *Auto-generated* atau *manual input* (Format: `01/JCS/INV/VII/2026`).
* **Tanggal:** *Date picker*.
* **Nama Student:** *Text input*.
* **Level:** *Dropdown* (P1/P2/K1/K2) atau *text input*.
* **Jatuh Tempo:** *Date picker*.

**C. Detail Tagihan (Dynamic Table)**
Sistem harus mengakomodasi penambahan baris tak terbatas dengan kolom:
* **Keterangan:** Nama biaya (Contoh: Registration Fee, Admission Fee).
* **Nominal:** Format mata uang (Rupiah).
* **Keterangan Status:** *Dropdown* (Lunas / Belum Lunas).

**D. Ringkasan Keuangan (Auto-Calculated)**
* **Total:** Penjumlahan seluruh *Nominal*.
* **Bayaran Diterima:** *Input* manual dari pengguna (uang yang sudah dibayar).
* **Sisa Tagihan:** Hasil dari `Total` dikurangi `Bayaran Diterima`.

**E. Informasi Tambahan (Footer)**
* **Pesan:** *Text area* untuk instruksi konfirmasi pembayaran (termasuk variabel nama siswa dan nomor WhatsApp).
* **Detail Pembayaran:** Nama Bank, Nomor Akun Bank, Atas Nama.
* **Tanda Tangan:** Nama manajer keuangan (Contoh: RR Ratih Retno Sari, S.P) dan Jabatan (Finance Manager).

### 4.3. UI/UX dan Templat PDF
* **Tampilan Pratinjau (Preview Mode):** Harus meniru persis struktur tata letak (layout), garis pembatas, penempatan teks (rata kiri/kanan), dan posisi logo yang ada pada referensi visual.
* **Ekspor PDF:** PDF yang dihasilkan harus memiliki margin yang rapi, berukuran kertas A4, dan *font* yang jelas dibaca saat dicetak fisik maupun dikirim melalui platform digital (seperti WhatsApp atau Email).

## 5. Struktur Database (Backend)

Sistem membutuhkan *database schema* dasar sebagai berikut:

* **`invoices` table:**
  * `id` (Primary Key)
  * `invoice_number` (String, Unique)
  * `date` (Date)
  * `due_date` (Date)
  * `student_name` (String)
  * `student_level` (String)
  * `total_amount` (Decimal)
  * `amount_received` (Decimal)
  * `remaining_balance` (Decimal)
  * `notes` (Text)
  * `created_at`, `updated_at` (Timestamps)
* **`invoice_items` table:**
  * `id` (Primary Key)
  * `invoice_id` (Foreign Key)
  * `description` (String)
  * `amount` (Decimal)
  * `status` (Enum/String: 'Lunas', 'Belum Lunas')
* *(Opsional)* **`settings` table:** Untuk menyimpan data instansi dan detail rekening bank agar tidak perlu diketik ulang setiap saat.

## 6. Kriteria Penerimaan (Acceptance Criteria)
1. Pengguna dapat membuat *invoice* baru dengan mengisi semua kolom tanpa ada *error* pada *frontend*.
2. Kalkulasi "Total" dan "Sisa Tagihan" berjalan otomatis secara *real-time* saat pengguna memasukkan angka.
3. Format angka otomatis dikonversi menjadi format mata uang lokal (misal: `Rp 20,000,000`).
4. Tombol "Download PDF" berhasil menghasilkan file `.pdf` yang memuat seluruh data.
5. Layout PDF harus sejajar secara presisi dengan tata letak visual referensi `image_1249e1.png`.
6. API Laravel terlindungi dan hanya dapat diakses melalui permintaan yang valid dari *frontend* React.
